Data Types#

RuneScape uses a number of uncommon and bespoke data formats to obsfuscate communication and reduce cache file size.

Each client revision randomises the endianness and modifications used for decoding client packets. The OSRS protocol scrambling follows a decipherable order (citation needed).

Standard Java Data Types#

RuneScape client uses Java's primitive data types to store and transmit information, understanding these limitations and sizes are important regardless of client revision.

Obfuscation methods#

Endianness#

The read and write order of byte data is called Endianness RuneScape client occasionally uses a middle-endian (integer's only) the order of which is determined by an inverse modification.

  • Big - Most significant bit first
  • Little - Least significant bit first
  • Middle - Most significant bit central

Bespoke types#

Medium/Tribyte#

A custom data type for storing 3 byte numbers between -8,388,608 and 8,388,607

Smart#

Functions that "smartly" choose the smallest data type necessary to store the value provided

e.g

if value < byte max
write byte
else
write short
